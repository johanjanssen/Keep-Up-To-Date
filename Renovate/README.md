# Gitea + Jenkins + Renovate — Local Demo

This directory contains everything needed to run the **HelloConference** live demo locally:
a self-hosted Gitea forge, a Jenkins CI server, and the Renovate bot — all wired together
in Docker, fully automated by five numbered scripts.

---

## Services & Credentials

| Service | Image | URL | Username | Password |
|---|---|---|---|---|
| **Gitea** | `gitea/gitea:1.22.3` | http://localhost:3000 | `gitadmin` | `Admin1234!` |
| **Jenkins** | custom (see `docker/jenkins/Dockerfile`) | http://localhost:8080 | `admin` | `Admin1234!` |
| **Renovate** | `renovate/renovate:latest` | one-shot container | — | — |

---

## Quick Start

```bash
# From the project root:
bash Renovate/scripts/demo.sh
```

**Total time:** ~10–15 minutes on first run (Jenkins image build + plugin download).  
**Subsequent runs:** ~3–5 minutes (image is cached).

---

## Step-by-Step

Run individual steps if you need to retry or inspect between stages:

```bash
bash Renovate/scripts/01-start-gitea.sh     # start Gitea, wait for readiness
bash Renovate/scripts/02-setup-gitea.sh     # create admin, repo, API token → writes Renovate/.env
bash Renovate/scripts/03-push-code.sh       # git push HelloConference to Gitea
bash Renovate/scripts/04-start-jenkins.sh   # build image, start Jenkins, create pipeline job
bash Renovate/scripts/05-run-renovate.sh    # run Renovate → creates real PRs
```

## What Happens

### Renovate PRs

When `05-run-renovate.sh` runs, Renovate scans the Gitea repository and opens:

| PR | Trigger | Rule in `renovate.json` |
|---|---|---|
| 🔴 `log4j-core 2.0 → 2.x` | **CVE-2021-44228** (Log4Shell, CVSS 10.0) | `vulnerabilityAlerts` — created immediately |
| 🟡 Spring Boot 4.1 → latest | Newer version available | grouped, scheduled weekends |
| 🟢 `eclipse-temurin` digest | OS-level patches in base image | `docker.pinDigests: true` |
| 🟢 `gcr.io/distroless` digest | OS-level patches | `docker.pinDigests: true` |
| 🟢 `gitea/gitea:1.22.3` | Newer Gitea release | standard version update |

### Jenkins Builds

Every PR triggers a Jenkins build automatically via Gitea webhook:

```
Renovate opens PR
  → Gitea fires webhook to http://jenkins:8080/gitea-webhook/post
      → Jenkins Multibranch Pipeline builds the PR branch
          → ./mvnw compile → test → package
              → JUnit results published
                  → Pass ✅ / Fail ❌ posted back to Gitea PR
```

The build result appears in the **Checks** section of each Gitea pull request.

## Directory Structure

```
Renovate/
├── .env                            ← generated by 02-setup-gitea.sh (gitignored)
│                                     contains: GITEA_TOKEN=<api-token>
├── docker/
│   ├── docker-compose.yml          Gitea + Jenkins services
│   └── jenkins/
│       ├── Dockerfile              Jenkins LTS + Docker CLI + pre-installed plugins
│       ├── plugins.txt             Plugin list (pin versions for Renovate to update)
│       ├── casc/
│       │   └── jenkins.yaml        JCasC — admin user, server URL
│       └── jobs/
│           └── hello-conference.xml  Multibranch Pipeline job definition
├── scripts/
│   ├── demo.sh                     master script — runs all five steps in order
│   ├── 01-start-gitea.sh           starts Gitea container, waits for readiness
│   ├── 02-setup-gitea.sh           creates admin user, repo, API token → Renovate/.env
│   ├── 03-push-code.sh             commits and force-pushes project to Gitea
│   ├── 04-start-jenkins.sh         builds Jenkins image, starts container, configures job
│   ├── 05-run-renovate.sh          runs Renovate bot → opens PRs, triggers Jenkins builds
│   └── reset-demo.sh               wipe everything for a clean re-run (run manually)
└── renovate.json                   per-repo Renovate config (source of truth)

# Generated at project root by 03-push-code.sh (gitignored):
renovate.json                       copy of Renovate/renovate.json — read by the Renovate bot
```

---

## Generated Files

`02-setup-gitea.sh` writes a single file that is **gitignored** and must never be committed:

| File | Contains | Used by |
|---|---|---|
| `Renovate/.env` | `GITEA_TOKEN=<api-token>` | scripts 03, 04, 05 |

To inspect the token: `grep GITEA_TOKEN Renovate/.env`

---

## Configuration Split

| File | Controls |
|---|---|
| `Renovate/renovate.json` (source) | Update groups, stability windows, auto-merge, vulnerability alerts |
| `renovate.json` (project root) | Generated copy — what the Renovate bot actually reads from Gitea |
| `docker/jenkins/casc/jenkins.yaml` | Jenkins admin user, server URL |
| `docker/jenkins/jobs/hello-conference.xml` | Multibranch Pipeline job definition |
| `Jenkinsfile` (project root) | What Jenkins builds for each branch/PR |

## Troubleshooting

```bash
# Gitea logs
docker compose -f Renovate/docker/docker-compose.yml logs -f gitea

# Jenkins logs (plugin loading, JCasC errors, build output)
docker compose -f Renovate/docker/docker-compose.yml logs -f jenkins

# Re-run step 2 only (re-creates the API token and refreshes Renovate/.env):
bash Renovate/scripts/02-setup-gitea.sh

# Re-run step 4 only (recreates the Jenkins job + Gitea credential):
bash Renovate/scripts/04-start-jenkins.sh

# Renovate/.env missing or token expired?
bash Renovate/scripts/02-setup-gitea.sh   # re-creates token and writes fresh .env
```

## Reset & Cleanup

```bash
# Stop containers but keep Gitea and Jenkins data (fast resume)
docker compose -f Renovate/docker/docker-compose.yml down

# Stop containers and delete all volume data
docker compose -f Renovate/docker/docker-compose.yml down -v

# Full reset — removes containers, volumes, Jenkins image, Renovate/.env,
# and the local 'gitea' git remote (clean slate for the next demo run):
bash Renovate/scripts/reset-demo.sh
```
